# 23. Plan de Gestión de Riesgos

**Metodología:** Identificación temprana durante el Sprint 0, actualización semanal en reuniones de sincronización.
- **Tolerancia:** Cero tolerancia a riesgos que comprometan la demostración final (ej. caída de hardware). Para mitigar esto se priorizó el Simulador de Software.
- **Responsabilidades:** El Director del Proyecto consolida el registro; cada líder técnico identifica riesgos en su área (ej. Ángel en ML, Luis en React).
- **Herramienta:** Actualización del archivo `24-registro-riesgos.md` en el repositorio.
